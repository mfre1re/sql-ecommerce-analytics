# Contexto e Objetivo

## Contexto

Este projeto simula o banco de dados de uma operação de e-commerce, contemplando entidades como clientes, produtos, pedidos, pagamentos e entregas.  
A proposta é construir uma base relacional realista para praticar SQL do nível básico ao intermediário, com foco em modelagem, integridade dos dados e análise orientada a negócio.

## Objetivo Geral

Desenvolver um projeto completo de banco de dados SQL, desde a modelagem até consultas analíticas, demonstrando domínio prático de estruturação relacional e interpretação de indicadores.

## Objetivos Específicos

1. Modelar o domínio em DER e implementar o esquema relacional com boas práticas.
2. Garantir consistência com PK, FK, `NOT NULL`, `UNIQUE` e `CHECK`.
3. Criar consultas para KPIs como faturamento, ticket médio, retenção e produtos mais vendidos.
4. Aplicar SQL intermediário com CTEs, subqueries e window functions.
5. Organizar scripts e documentação reprodutível para portfólio técnico.
